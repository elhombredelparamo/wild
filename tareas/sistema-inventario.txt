# SISTEMA DE INVENTARIO DEL JUGADOR

## Fecha
2026-03-06 - 12:03

## Descripción
Implementar sistema completo de inventario para gestión de objetos del jugador.

## Características básicas
- **Espacio de inventario:** Grid de slots (ej: 5x9 = 45 slots)
- **Apilamiento:** Items apilables con límites por tipo
- **Drag & drop:** Arrastrar items entre slots
- **Quick access:** Barra de acceso rápido (1-9)
- **Ordenamiento:** Auto-ordenar por tipo/nombre

## UI del inventario
- **Ventana modal:** Abrir/cerrar con tecla (ej: I)
- **Visualización:** Iconos de items + cantidad
- **Información:** Tooltip con detalles al pasar mouse
- **Categorías:** Filtros por tipo de item
- **Búsqueda:** Buscar items por nombre

## Sistema de items
- **Item base:** Clase abstracta para todos los items
- **Tipos de items:**
  - Herramientas (pico, hacha, etc.)
  - Armas
  - Materiales (madera, piedra, etc.)
  - Comida
  - Armadura
  - Misceláneos
- **Propiedades:** Nombre, descripción, peso, valor, durabilidad

## Integración con sistemas existentes
- **PlayerController:** Acceso a inventario
- **GameFlow:** Guardar/cargar inventario
- **NetworkManager:** Sincronización en multijugador
- **UI System:** Integración con interfaz existente
- **InputSystem:** Manejo de teclas de acceso rápido

## Componentes técnicos
- **Inventory.cs:** Gestión principal del inventario
- **InventorySlot.cs:** Slot individual con item y cantidad
- **Item.cs:** Clase base para todos los items
- **InventoryUI.cs:** Interfaz visual del inventario
- **ItemDatabase.cs:** Registro de todos los tipos de items

## Mecánicas avanzadas
- **Capacidad de peso:** Límite de peso total
- **Durabilidad:** Items que se desgastan con uso
- **Calidad:** Diferentes niveles de calidad
- **Sets bonificación:** Conjuntos de items con bonificaciones
- **Encantamientos:** Mejoras mágicas futuras

## Interacciones
- **Recoger items:** Del terreno, cofres, mobs
- **Tirar items:** Soltar al mundo
- **Transferir:** Entre inventarios (cofres, otros jugadores)
- **Consumir:** Comida, pociones
- **Equipar:** Armadura, herramientas

## Persistencia
- **Guardado automático:** Con estado del jugador
- **Formato eficiente:** JSON o binario optimizado
- **Versioning:** Compatible con futuras expansiones
- **Validación:** Prevenir cheats/modificaciones

## Variables configurables
- **Tamaño inventario:** Filas x columnas
- **Límite peso:** Máximo peso transportable
- **Auto-orden:** Activar/desactivar
- **Quick slots:** Número de slots rápidos

## Prioridad
Alta - Fundamental para mecánicas de juego

## Notas
- Base para sistemas de crafteo y construcción
- Esencial para progresión del jugador
- Requiere balance cuidadoso de capacidades
- Escalable para futuras expansiones
- Considerar accesibilidad para diferentes dispositivos
